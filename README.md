# Automatyczna stacja pogodowa
Repozytorium pozwalające dokumentować projekt automatycznej stacji pogodowej na POS

Osoba odpowiedzialna za utrzymanie porządku: Hubert Wilarski

Projekt podzielony jest na 4 główne etapy realizacji:
1. Identyfikacja klienta, użytkowników końcowych i innych zainteresowanych stron tematu (stakeholders); zgromadzenie i przeanalizowanie szczegółowych wymagań funkcjonalnych i pozafunkcjonalnych; określenie scenariuszy użycia i opracowanie specyfikacji funkcjonalnej z diagramami UML (komponentów, klas, aktywności, inne);
2. Uzyskanie dostępu do repozytorium systemu kontroli wersji i śledzenia błędów – np. zewnętrzny hosting (GitHub, Bitbucket…) oraz wyznaczenie osoby odpowiedzialnej za utrzymanie porządku w repozytorium (maintainer); zaplanowanie struktury repozytorium uwzględniając wybrane używane środowisko IDE, język programowania, dodatkowe artefakty (grafiki UI, specyfikacje itp.). Opracowanie modeli UML kluczowych elementów projektowanego rozwiązania; analiza możliwości użycia dodatkowych bibliotek i narzędzi do realizacji zadania; opracowanie specyfikacji technicznej podsumowującej te informacje wraz z harmonogramem i podziałem pracy w zespole.
3. Tworzenie dokumentacji HTML z komentarzy Doxygen. Podczas implementacji kodu programów zaangażować odpowiedni nakład pracy na dokumentowanie API (komentarze zgodne z Doxygen) oraz inne komentarze w kodzie, wyjaśniające szczególnie zagmatwane algorytmy, mechanizmy, zależności.
4. Testowanie przygotowanego oprogramowania. Wykryte podczas cykli testowania błędy raportować do systemu śledzenia błędów; opcjonalnie zastosować metodę test-driven development do zapewnienia jakości; zademonstrować użycie testów jednostkowych do walidacji kluczowych elementów oprogramowania. Użyć profilera do wykrycia hot spotów w opracowanym programie i przeprowadzić próbę ich optymalizacji.
